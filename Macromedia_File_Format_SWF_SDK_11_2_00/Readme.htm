Macromedia Flash File Format SDK – Read me

 


The Macromedia Flash File Format (SWF) SDK provides a set of tools for developers to write Macromedia Flash (SWF) version 5 files. The Macromedia Flash File Format (SWF) SDK is comprised of a low level interface and a high level interface, both of which write Macromedia Flash (SWF) files, but are intended to be used in different ways.

Installation and Executing

The project is provided ready to compile on a Windows systems. A Visual C++ project file is provided.

 

1.        Copy the Source directory to your hard disk.

2.        Open the Workspace file, swfproduce.dsw with Visual C++.

3.        Build All

4.        Now you should be able to run swfproduce.exe directly from the environment.

 

Note that if you run from the command line, the bm128.jpg and audio.wav files need to be in the current directory. This should not be a problem if you run from the Visual C++ IDE.

 

Execution will briefly bring up a console app which should run and quickly complete. The app will write a series of sample Macromedia Flash (SWF) files to the local directory. You can then view the sample . Macromedia Flash (SWF) files and the example .cpp files that created them, which have the same base name. For instant Macromedia Flash (SWF) gratification, try looking HFExample.swf.

 

There will be 2 sets of examples written to your disk:

·         HFExample*, examples created with the high level manager.

·         FExample*, examples created with the low level manager.

Documentation

The Macromedia Flash File Format (SWF) SDK Documentation is provided in both HTML and Microsoft Word formats. Documentation is provided for:

·         The Macromedia Flash (SWF) File Format, explaining how the file format works.

·         The Macromedia Flash (SWF) Manager, which gives programmers an API with which to write Macromedia Flash (SWF) files.

 

Additional notes about the documentation can be found in the Documentation subdirectory.

 

The first 2 subdirectories below Macromedia_Flash_File_Format_(SWF)_SDK are HTML and Microsoft Word. The entire documentation is echoed in both formats. (Except the API_Docs which are only available in HTML format, but the HTML files are in the Microsoft Word tree for completeness.)

 

The Macromedia Flash File Format (SWF) _Documentation directory contains an explanation of the Macromedia Flash File Format (SWF). The file “intro.doc/html” provides a basic introduction and links to the individual Macromedia Flash (SWF) topics.

 

The Macromedia Flash (SWF) _Manager_Docs directory contains the “SDK.doc/html” file which describes the basics of using the high and low level manager to create Macromedia Flash (SWF) files. The subdirectory API_Docs is a reference (in html) to the high and low level API. The *_files subdirectories contain images for the html docs, and should be ignored.

Using the Macromedia Flash File Format (SWF) SDK in other programs

1.        Add all the Macromedia File Format (SWF) SDK files to your project, including the subdirectories PCM and ZLib-1.1.3.

2.        Remove all example files HFExample*.* and FExample*.*

3.        Remove swfproduce.cpp (contains main() )

4.        Include the HF3SDK header in your files that use the high level SDK, and the F3SDK header for files that use the low level Macromedia Flash File Format (SWF) SDK.

The Low Level Interface

The low level interface can write Macromedia Flash (SWF) files of any type. Every Macromedia Flash (SWF) 5 type (covered in the Macromedia Flash File Format (SWF) Documentation) is represented by a class in the low level manager. Classes that begin with ‘F’ are low level classes. Examples of the low level interface are in the FExample* files.  You would use the low level interface if you understand Macromedia Flash (SWF) and wish to write Macromedia Flash (SWF) tags directly, or if you are trying to write unusual Macromedia Flash (SWF) files.

 

Documentation for the low level classes is provided in hyper linked html, located in the “API_Documentation” subdirectory. It is in Java doc format.

 

The low level manager contains many examples, for which both the SWF file and the .cpp file have the FExample* prefix.

 

To examine some sample code, the best place to start is FExampleBitsLossless.cpp. The lossless bitmap example has the most detailed documentation.

The High Level Interface

The high level interface is used to write Macromedia Flash (SWF) in a more natural way. The developer can add Rectangle, Circle, Bitmap, EditText, and other primitive objects to Frames in a Movie. The Movie can then be serialized to disk. It's an easy, fast way to write Macromedia Flash (SWF).

 

High level classes begin with ‘HF’. Examples of the high level interface are in the HFExample* files.

 

Documentation for the high level classes is provided in hyper linked html, located in the “API_Documentation” subdirectory.

 

To review some sample high level code, the best place to start is HFCircle.cpp, since this is the example easiest to understand. HFExample.cpp is also worthwhile to examine; while much larger, it demonstrates all of the features included in the high level manager.
