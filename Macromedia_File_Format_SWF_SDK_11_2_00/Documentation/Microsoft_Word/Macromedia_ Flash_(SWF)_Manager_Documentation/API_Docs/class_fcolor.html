Detailed Description

This object specifies a color object.

FColor is used (copied) as a structure. Do not add dynamic memory or virtual functions.

There are two types of FColor, i.e. one with alpha channel on and the other with alpha channel.off. When you construct an instance of FColor, if you provide three parameters, the alpha channel is off; if you provide four, the alpha channel is on.

Each color-related SWF tag requires the correct color type, either RGB or RGBA. Sometimes, For example, DefineShape, DefineShape2, SetBackgroundColor need RGB (alpha off); DefineShape3, DefineMorphShape, DefineEditText need RGBA (alpha on).

The low-level manager can correct some errors of wrong color type, but not all. So don't rely on it. If the wrong one got over low-level manager, the flash play might crash on that.

We recommend you use RGBA whenever reasonable. The reason? Using RGBA is very consistent and worth the sacrifice of some file space (in SWF, it's one more byte for each occurance of color info, and some additional for color transformation.)

Actually, we only provide RGBA interface for colors in high-level manager because this solution is much more elegant in terms of consistency and compatibility.
