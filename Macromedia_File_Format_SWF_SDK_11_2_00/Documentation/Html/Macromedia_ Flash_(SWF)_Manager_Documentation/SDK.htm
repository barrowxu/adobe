Using the Macromedia Flash File Format (SWF) SDK

 

The Macromedia Flash File Format (SWF) SDK is a C++ Library that provides all the functionality required to create Macromedia Flash (SWF) files.  It is divided into two sections:

 

1.       The Low Level Manager

2.       The High Level Manager

 

The Low Level Manager

The Low Level Manager has been designed to follow the structure of Macromedia Flash (SWF) as closely as possible.  It includes a set of C++ classes that mirror the tag structure of Macromedia Flash (SWF).  There is a C++ class for each tag that Macromedia Flash (SWF) defines.  The parent of each of these classes is FObj (for Flash Object).   Sometime we use an abstract class when several FObjs are closely related to each other, for example similar tags that changed only slightly between Macromedia Flash versions.

 

A FObjCollection is simply a list that contains all the FObjs in a Macromedia Flash (SWF) file.

 

Low Level Classes

Here is a breakdown of the basic classes in the Macromedia Flash File Format (SWF) SDK:

 

The Movie Object

The overall container for a Macromedia Flash movie
FObjCollection

Holds the collection of Tag objects until it’s time to write to disk.

FObj

Control Tag Classes

C++ Objects for Control Tags that place, move, and modify a Data Tag on the Display List.  Control classes have names beginning with “FCT”

 

Data Tag Classes

C++ Objects for Data Tags that store the basic graphics, text, and sound objects that comprise Macromedia Flash objects.  Data classes all have names beginning with “FDT”

 

 
Primitive Data Classes

Data Packing Primitives hold the “guts” of the Data Tags.  Data Primitives know how to write (serialize) themselves to a data stream so they save you from having to know how about compression and byte swapping.   You add instances of these classes to the Control and Data Classes.
Library Classes

Helper libraries for sound and bitmaps

 

 

Low Level Memory Management

The low level manager takes over ownership of the objects passed to it. So in the following example:

 

FCTSetBackgroundColor* background;

background = new FCTSetBackgroundColor(white);

    allTags.AddFObj(background);

 

The memory pointed to by ‘background’ will be deleted by the low level manager when it writes the file stream. After ‘background’ is passed to the low level manager, it should be considered lost to the caller.

 

Some large objects are not deleted, owned, or modified by the low level manager. Specifically bitmaps and sounds, because of their size, are left in the control of the calling routines. The pointer to objects not deleted or modified by the low level manager are declared const.

What’s it like to create a simple Macromedia Flash (SWF) file with the LLM?

 

The structure of a movie being created using the Macromedia Flash File Format (SWF) SDK is very similar to the structure of an Macromedia Flash (SWF) file.  Here are the steps:

 

  • Create Movie
  • Create an FObjCollection for Control and Data Objects
    1. Create Data Tag Object
    2. Create a data Primitive
    3. Add the Primitive to Data Tag Object
    4. Add Data Object to Collection
    5. Create Control Object
    6. Add Control Object to Collection
  • repeat 1 through 6
  • Write FObjCollection

 

Low Level Manager Example

 

Here is an example of a routine we’ve created that creates a simple movie.  The movie simply creates a red rectangle.  Don’t worry about the particulars of each routine, simply look at the overall flow. 

 

 

#include "F3SDK.h"     

#include "FExample.h"         //just the prototypes for our examples

 

Our routine creates a one frame movie: a rectangle with a red fill color, a standard 1 pixel wide,

black line style

 

void CreateRectangleMovie(){

 

Create a collection of FObj's, allTags, to contain the FObjs that make the movie

Each Macromedia Flash (SWF) tag in the movie will be represented by an FObj

 

    FObjCollection allTags;

 

Construct first flash tag object (set background color):

 

Define a color for the background

 

    const FColor white(0xff, 0xff, 0xff);

 

Construct the SetBackgroundColor object which takes a color as an argument

All routines beginning with FCT create Macromedia Flash Control Tags

 

FCTSetBackgroundColor* background;

background = new FCTSetBackgroundColor(white);

 

Add the SetBackgroundColor tag to allTags, the FObjCollection

 

    allTags.AddFObj(background);

 

 

Now start creating the rectangle object. You must:

1.       Create the bounds Rect

2.       Create the DefineShape FObj and remember its ID for use in the PlaceObject Tag

3.       Create the Color for the fill

4.       Create the Fill itself and pass it the color

5.       Add the fill to the shape

6.       Create the color for the rectangle's line style

7.       Create the LineStyle record

8.       Create the Edge records that define the rectangle and add them to the shape

 

9.       construct a rect that defines the shape's bounds

 

Create the bounding rectangle (in TWIPs)

 

    FRect* rectBounds = new FRect(1000, 5000, 1000, 5000);

 

Construct the FDTDefineShape which will be the rectangle image

 

    FDTDefineShape* rectangle = new FDTDefineShape(rectBounds);

 

Record its ID so that we can later refer to it when we place it on the display list

 

    U16 rectangleID = rectangle->ID();

 

Construct a red Fcolor to fill the rectangle. Note that since a FDTDefineShape was used, we have to use an FColor without an alpha channel.

    FColor red = FColor(0xff, 0, 0);

   

Use the Red FColor to create a FillStyle record

    FFillStyle* fill = new FFillStyleSolid(red);

 

Add the fill style to the rectangle. You must record the position of the fill style in the fill style array (the ID) so you can tell Flash when to use it. 

 

    U32 redfillID = rectangle->AddFillStyle(fill);

   

Construct a black color

 

    FColor black = FColor(0, 0, 0);

 

Construct a black, 1 pixel (20 TWIPS) wide line style

 

    FLineStyle* blackLineStyle = new FLineStyle(20, black);

 

Add the line style to the rectangle FObj, remembering to store the position of the line style just as in the fill style

 

    U32 blackLineStyleID = rectangle->AddLineStyle(blackLineStyle);

 

Since you are done creating fill and line styles, tell the LL manager so.

 

    rectangle->FinishStyleArrays();

 

Construct the shape records which will describe the rectangle.

The FShapeRecChange sets the X-Y position of the initial line. (All points in Macromedia Flash are relative, like Postscripts lineto.)  FShapeRecEdgeStraight is used to create the 4 lines of the rect, and FShapeRecEnd finishes it off.

 

    FShapeRec* rectangleShapeRecords[6];

    rectangleShapeRecords[0] = new FShapeRecChange(false, true, true, false, true, 5000, 1000, NULL, redfillID, blackLineStyleID, NULL, NULL);

 

Create straight edge objects

 

    rectangleShapeRecords[1] = new FShapeRecEdgeStraight( 0, 4000);

    rectangleShapeRecords[2] = new FShapeRecEdgeStraight( -4000, 0);

    rectangleShapeRecords[3] = new FShapeRecEdgeStraight( 0, -4000);

    rectangleShapeRecords[4] = new FShapeRecEdgeStraight( 4000, 0);

    rectangleShapeRecords[5] = new FShapeRecEnd();

   

Add the shape records to the rectangle shape object

 

    for (int i = 0;  i < 6 ;  i++)

          rectangle->AddShapeRec(rectangleShapeRecords[i]);

 

Add the rectangle to the allTags object collection

 

    allTags.AddFObj(rectangle);

 

Create a place object tag which puts the rectangle on the display list

 

    FCTPlaceObject2 *placeRectangle = new FCTPlaceObject2(false, false,false, false, true, false, 1, rectangleID, NULL, NULL, NULL, NULL);

 

Add the place object tag to the FObjCollection

    allTags.AddFObj(placeRectangle);

 

Construct a show frame object

    FCTShowFrame *showFrame = new FCTShowFrame();

 

Add the show frame object to the FObj collection;

    allTags.AddFObj(showFrame);

 

Now create the movie

    allTags.CreateMovie("rectangle.swf", 11000, 8000, 12);

 

}

The High Level Manager

 

The High Level Manager provides a higher level of abstraction for creating Macromedia Flash (SWF) files.  High level classes begin with the prefix  “HF”.  There are HF classes for creating movies, frames, circles, rectangles, text and bitmaps.  High-level classes are much easier to use than low-level classes.  For example, the HFCircle class creates a circle using a center point, and radius.  There is no need to construct the circle with curved shape records.

 

High Level Classes

 

HFMovie – High-level SWF movie.

HFFrame – Frame in a high-level movie.

HFObject – Parent class for all high-level characters.

HFSound – High-level sound.

HFButton – High-level button.

HFShape – Parent class for all high-level shapes. 

HFRectangle – High-level circle.

HFBitmap – High-level rectangle filled with bitmap

HFCircle – High-level circle.

HFPolygon – High-level polygon.

HFOval – High-level oval (or ellipse).

HFFont – High-level static text.

HFText – High-level static text.

HFEditText – High-level editable text field.

HFAction – High-level action.

High Level Manager Resource Management

Memory

The high level manager copies the information from the objects passed to it. Sometimes it will modify the objects passed to it, but the high level manager will not delete them.. Objects passed to the high level manager can be created on the stack.

 

The memory model for the high level manager is almost the opposite of the low level manager.

 

Consider the following example:

 

HFMovie movie;

{

      HFRectangle rect( 0, 0, 200, 200 );

      movie.Frame( 0 )->AddObject( &rect );

}

movie.WriteMovie("wierd.swf");

   

The rectangle is written to the file without error, even though the object itself is out of scope. (This example is pedagogical in its structure – there is no need for the Rectangle object to have a smaller scope than the HFMovie object.)

Frames

Frames are allocated by the high level manager on an “as needed” basis when they are accessed with the Movie::Frame( int ) method call.

Depth

The Macromedia Flash Player uniquely identifies shapes by a tag (associated with its Define tag) and a depth. So two instances of the same object need to be on different depths to be unique. The high level manager hides this limitation and guarantees correct depths by writing each shape to a different depth in the Macromedia Flash (SWF) file.

 

The result is that the depth specified in the SetDepth call will correctly set a shape’s sorting order relative to other shapes, but this value will not be the depth written to the Macromedia Flash (SWF) file. Shapes set to the same depth with SetDepth have an undefined sorting order.

 

To summarize: higher depths are on top, and shapes with the same depths sort in an undefined order.

High Level Manager Example

 

Here is an example of using the high-level manager to create a simple movie.  This code creates a red rectangle and blue rectangle, places them on the display list, and displays the frame.

 

Include high-level class definitions

 

#include "HF3SDK.h"

 

void HLCreateRectangleMovie()

{

 

First, construct a high-level movie object.

 

    HFMovie movie;

 

Create a rectangle with the top-left corner at (0,0) and the bottom-right at (2000,2000)

 

    HFRectangle* rect1 = new HFRectangle( 0, 0, 2000, 2000 );

 

Create another rectangle with the top-left corner at (1000,1000) and bottom-right at (3000,3000)

 

    HFRectangle* rect2 = new HFRectangle( 1000, 1000, 3000, 3000 );

 

Set the fill color of the first rectangle to red.

 

    rect1->SetSolidFillColor( Red_RGBA );

 

Set the fill color of the second rectangle to blue.

 

    rect2->SetSolidFillColor( Blue_RGBA );

 

Set the depth of the first rectangle to 1 (the bottom of the stack).

 

    rect1->SetDepth(1);

 

Set the depth of the second rectangle to 2 (above the first rectangle).

 

    rect2->SetDepth(2);

   

Add rect1 and rect2 to frame1, then place them on frame1

movie.Frame( 0 )->AddObject( rect1 );

movie.Frame( 0 )->AddObject( rect2 );

   

Write the movie to the SWF file.

 

    movie.WriteMovie("HLrectangle.swf");

 

Clean up the memory.

    delete rect1;

    delete rect2;

}
